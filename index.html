Accessibility

WCAG 2.x contrast ratios for all key color pairings. Target: AA (4.5:1 normal text · 3:1 large text and UI components).

Pairing Ratio Level Context
Dark product surfaces
Cream on Warm Dark Base 16.32:1 AAA Primary body text
Cream on Bar 15.66:1 AAA TopBar / SubBar text
Cream on Card 14.38:1 AAA Card body text
Gold on Warm Dark Base 10.98:1 AAA Owner badge — high contrast on dark (10.98:1), low on light (1.73:1)
Label on Warm Dark Base 10.26:1 AAA Subtitle / label text
Copper on Warm Dark Base 5.38:1 AA Active tab, selected state
Tertiary on Warm Dark Base 4.94:1 AA Meta / caption text
Error Red on Warm Dark Base 5.06:1 AA Inline error on dark
Placeholder on Warm Dark Base 3.07:1 AA UI Input placeholder — UI component threshold
Admin Teal on Warm Dark Base 2.95:1 Exception Badge indicator only, never body text — see notes below
Disabled on Warm Dark Base 1.90:1 By design Intentionally inaccessible — signals non-interactivity
Light surfaces — auth / marketing
Near-black on White 17.40:1 AAA Primary text, light surfaces
Secondary text on White 5.74:1 AA Supporting text, light surfaces
Admin Teal on White 6.45:1 AA Admin badge on light surface
White on Copper (button) 3.53:1 Exception Primary CTA — see notes below
Error Red on White 3.76:1 Exception Inline error messages — see notes below
Gold on White 1.73:1 By design Low contrast as text (1.73:1). Acceptable as a graphical badge at sufficient size.
Known exceptions — rationale
Exception White on Copper · 3.53:1 · Primary CTA button text

The WCAG 2.x luminance formula under-represents perceptual contrast for saturated warm hues. Hue separation between white and copper reads as higher contrast than the ratio alone suggests. Button text is rendered at 15px/600 weight with clear interactive affordance. Expected to pass under WCAG 3.0 / APCA. Do not darken copper to chase this number — it would compromise the brand identity.

Exception Admin Teal on Warm Dark Base · 2.95:1 · Role badge indicator

Admin teal is used exclusively as a graphical badge dot on dark surfaces, never as body text. The 3:1 UI component threshold applies; at 2.95:1 it is marginally below. Revisit if admin badge ever carries text content.

Exception Error Red on White · 3.76:1 · Inline form error messages

Error red passes AA for large/bold text (3:1) but not normal body text (4.5:1). Inline errors are always prefixed with "Error:" per content guidelines and rendered in context where red color reinforces severity — never as the sole indicator of state. Acceptable as a compound signal.
